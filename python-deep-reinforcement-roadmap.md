# Python 深度强化训练总路线

这条路线用于在进入 LLM SDK 前，重新补强 Python 的深度、广度和工程精确性。

训练规则：

1. 每一课先给学习文档。
2. 学习者学完后说：`我学习完了`。
3. 然后出测试题。
4. 测试题包含概念题、代码判断题、改错题和小编码题。
5. 批改时直接指出错误、风险和工程后果。
6. 不因为“思路大概对”就通过；代码必须能运行，字段名、大小写、导入、缩进、返回格式必须精确。
7. 如果不通过，先补强，不进入下一课。

---

## 训练目标

完成这条路线后，学习者需要具备：

- 能准确解释 Python 代码运行过程；
- 能写出可运行、结构清晰、命名准确的 Python 代码；
- 能区分 `dict`、普通 `class`、`dataclass`、`BaseModel` 的使用场景；
- 能独立处理文件、JSON、异常、模块拆分；
- 能理解 OOP、类型提示、异步、FastAPI 的真实工程意义；
- 能为后续 LLM SDK、工具调用、RAG、Agent 工作流打下可靠基础。

---

## 总路线

### 第 1 课：Python 运行模型、变量、类型、对象与精确性

重点：

- Python 代码如何从上到下执行；
- 变量名、模块名、大小写敏感；
- `True` / `False` / `None`；
- 类型不是装饰，类型会影响运行逻辑；
- “定义类”和“创建对象”的区别；
- 为什么拼写错误会导致程序无法启动。

### 第 2 课：数据结构与 JSON 思维

重点：

- `list`、`dict`、`tuple`、`set`；
- 嵌套数据结构；
- JSON 与 Python 对象互转；
- 字段名精确性；
- Agent 应用为什么依赖结构化数据。

### 第 3 课：函数、参数、返回值与模块化

重点：

- `def`、参数、默认值、返回值；
- 函数边界；
- 为什么不要把所有逻辑写进一个函数；
- 函数如何变成 Agent 工具的基础。

### 第 4 课：文件、路径、环境变量与配置

重点：

- `open()`、`with`；
- `pathlib.Path`；
- JSON 文件读写；
- `.env` 和环境变量；
- API Key 为什么不能写死。

### 第 5 课：异常处理与工程错误意识

重点：

- `try...except`；
- 抛出异常与捕获异常；
- FastAPI 的 `HTTPException`；
- 什么时候应该失败，什么时候应该兜底；
- 错误信息如何帮助调试。

### 第 6 课：class、对象、dataclass、BaseModel

重点：

- `class`、`__init__`、`self`；
- 对象创建不用 `new`；
- `dataclass` 用于内部数据；
- `BaseModel` 用于 API / JSON 校验；
- 字段声明和赋值的区别。

### 第 7 课：类型提示与数据约束

重点：

- `str`、`int`、`bool`；
- `list[str]`、`dict[str, int]`；
- `Optional`、`Literal`、`Union`；
- `TypedDict`、`Protocol`；
- 类型提示如何提升可维护性。

### 第 8 课：OOP 工程化

重点：

- 继承；
- `super()`；
- 方法重写；
- `@property`；
- `@staticmethod` / `@classmethod`；
- 抽象基类。

### 第 9 课：异步编程

重点：

- `async def`；
- `await`；
- coroutine / task；
- `asyncio.gather()`；
- 超时、异常、并发控制；
- 为什么 LLM / API / RAG 场景常用异步。

### 第 10 课：FastAPI 工程基础

重点：

- 请求体；
- 响应模型；
- 路由；
- 状态码；
- 请求参数；
- 错误处理；
- 前端如何调用 Python 服务。

### 第 11 课：测试、调试与代码质量

重点：

- 如何读报错；
- 如何缩小问题范围；
- 简单单元测试；
- 命名、缩进、导入、返回值检查；
- 代码能跑才算完成。

### 第 12 课：Python 小项目实战

重点：

- 用 FastAPI 做一个小型任务分析服务；
- 支持 JSON 输入；
- 做参数校验；
- 做错误返回；
- 保存简单历史记录；
- 为后续 LLM SDK 替换核心逻辑做准备。
