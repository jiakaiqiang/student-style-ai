# 20 小时 Python 学习计划（前端转全栈版）

先说一个关键前提：**你已经是前端开发**，意味着变量、循环、函数、条件、数组对象、JSON、异步、API 请求这些概念你早就懂了。所以这 20 小时**不是从零学编程**，而是「把你脑子里的 JS 概念翻译成 Python，再补上后端那部分」。这能帮你省掉一半时间。

下面按优先级排，你可以直接照着做。

---

## 1. 一开始最该专注学什么（前 6 小时，最高优先级）

核心目标：**能流畅地用 Python 写出你已经会用 JS 写的东西。**

| 优先级 | 内容 | 对应你已会的 JS 概念 | 重点差异 |
|---|---|---|---|
| ★★★ | 缩进与代码块 | `{}` 大括号 | Python 用**缩进**代替 `{}`，这是你最大的坑（见第 3 节） |
| ★★★ | 变量与基本类型 | `let/const` | 不用声明关键字，直接 `x = 1` |
| ★★★ | 列表 / 字典 | 数组 / 对象 | `list` ≈ Array，`dict` ≈ Object |
| ★★★ | 函数 `def` | `function` | 默认参数、关键字参数比 JS 强大 |
| ★★★ | 条件 / 循环 | `if / for` | `for x in list:` 是默认写法，没有 C 式 for |
| ★★ | f-string 字符串 | 模板字符串 `` `${}` `` | `f"hello {name}"` |
| ★★ | 列表推导式 | `.map() / .filter()` | `[x*2 for x in nums]` 一定要练熟 |

**这 6 小时只做一件事**：把上面每个点写 3-5 个小例子，跑通。不要看视频看一整天，要**动手敲**。

---

## 2. 哪些内容可以暂时忽略（20 小时内别碰）

这些是新手最容易陷进去浪费时间的地方：

- ❌ **元类（metaclass）、装饰器原理、描述符** — 高级特性，用到再学
- ❌ **多线程 / 多进程 / GIL 细节** — 后端用 async 更常见，先放
- ❌ **`*args / **kwargs` 的所有花式用法** — 知道存在即可
- ❌ **Python 2 的任何内容** — 已死，别看老教程
- ❌ **复杂的 OOP 继承体系 / 设计模式** — 后端框架里用得少，先会写一个普通 class 即可
- ❌ **虚拟环境的各种工具对比**（conda vs venv vs poetry vs pdm） — 先只用 `venv` 一种，别纠结
- ❌ **类型注解的高级玩法**（泛型、Protocol） — 基础注解 `def f(x: int) -> str:` 够用了

> 原则：**20 小时内，凡是"原理性""对比性"的内容一律跳过，只学"能跑起来"的部分。**

---

## 3. 新手最容易犯的错误（尤其前端转过来的）

这一节请重点看，能帮你避开 80% 的卡壳：

1. **缩进错误（头号杀手）** 
   ```text
   顶层代码：0 个空格
   函数/类/if/for 内部：4 个空格
   同一个块：缩进必须完全一致（不要混用 Tab 和空格）
   ```
   你写惯了 `{}`，会下意识忘记缩进或缩进对不齐 → 直接报 `IndentationError`。

2. **冒号忘了写** — `if x > 1:` `def f():` `for i in x:` 后面都要 `:`

3. **用 JS 思维写 `===` / `&&` / `||`** — Python 是 `==` / `and` / `or` / `not`

4. **可变默认参数陷阱** — `def f(items=[])` 是经典坑，别用可变对象当默认值

5. **`for` 循环里改变量作用域的误解** — Python 没有块级作用域，循环变量循环结束后还在

6. **拷贝 = 引用** — `b = a`（list/dict）是引用，改 b 会改 a，要 `a.copy()`

7. **导入路径混乱** — `import` 机制和前端的 `import` 不一样，包/模块概念要花点时间理清

8. **忘记激活虚拟环境** 就 `pip install`，把包装到全局了

## 4. 每个阶段练什么（20 小时分配）

我把 20 小时切成 4 个阶段，每个阶段都有**明确的练手产物**：

### 阶段 A：语法翻译（6h）
**练**：写 10 个小脚本，每个 10-20 行
- 统计一个列表里每个单词出现次数（练 dict）
- 把一个 list 过滤+转换（练列表推导式）
- 写一个计算器函数（练 def + 参数）
- 读一段文本，按行处理（练字符串）

✅ 产物：10 个能跑的小 `.py` 文件

### 阶段 B：实用脚本 + 文件/JSON（5h）
**练**：做点真实有用的事（这也是你转全栈最快见效的部分）
- 读写文件（`open`、`with`）
- 解析 / 生成 JSON（`json` 模块，你前端最熟）
- 用 `requests` 库请求一个公开 API 并打印结果
- 写一个小脚本：批量重命名文件 / 整理文件夹

✅ 产物：一个「调用某个公开 API 并保存成 JSON 文件」的脚本

### 阶段 C：OOP 基础 + 异步入门（4h）
**练**：够用就行，别深入
- 写一个 `class`（属性、方法、`__init__`）
- 理解 `self`（≈ JS 的 `this`，但必须显式写）
- 写一个 `async def` + `await`（概念和 JS Promise/async 几乎一样）

✅ 产物：用 class 封装阶段 B 的 API 脚本

### 阶段 D：后端服务入门（5h）— 全栈的关键一跃
**练**：用 **FastAPI**（现代、文档好、对前端友好）
- 跑起一个 `Hello World` 接口
- 写一个 `GET` 和 `POST` 接口，返回 JSON
- 用 Pydantic 定义请求/响应数据结构（≈ TypeScript 的 interface，你会很有亲切感）
- 在浏览器打开自动生成的 `/docs` 接口文档

✅ 产物：一个有 3-4 个接口的本地 API 服务，前端能 fetch 它

> 为什么选 FastAPI 而不是 Django/Flask：对前端最友好、自带 API 文档、异步原生、和你未来做全栈/AI 应用衔接最好。

## 5. 如何判断自己已经入门（验收标准）

不用考试，用**能不能做出东西**来判断。满足下面这些就算入门了：

- [ ] 看到一段 Python 代码，能读懂它在干嘛，**不会被缩进/语法吓到**
- [ ] 不查资料能独立写出：读文件 → 处理数据 → 输出结果 的脚本
- [ ] 能熟练用 `list` / `dict` 和列表推导式处理数据
- [ ] 能用 `requests` 调一个 API，能用 `json` 解析返回
- [ ] 能写出一个简单的 `class` 并实例化使用
- [ ] **能用 FastAPI 起一个本地服务，前端页面能成功 fetch 到数据**
- [ ] 遇到报错，能看懂 Traceback 最后一行，自己去搜并解决
- [ ] 知道怎么建虚拟环境、装包、跑脚本

> **终极验收题（做出来就毕业）**： 
> 写一个 FastAPI 后端，提供一个 `/todos` 接口（增删查），数据存在一个 JSON 文件里；再用你熟悉的前端页面去调用它，完成一个最小全栈 Todo 应用。 
> 能独立做出这个，你的 Python 后端就**入门了**。

## 执行建议（怎么用好这 20 小时）

1. **别囤教程**。每个知识点学 10 分钟，立刻动手敲 20 分钟。
2. **用你前端已有的项目当练习场**。比如给你某个前端项目写个 Python 后端接口。
3. **报错是好事**。每个报错都让你更熟悉 Python，不要怕红字。
4. **20 小时建议节奏**：每天 2 小时 × 10 天，比一次性周末突击效果好。
